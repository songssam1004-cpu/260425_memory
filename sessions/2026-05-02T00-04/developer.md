### 🚀 Git Push를 위한 로컬 환경 전제 조건

로컬 환경에서 원격 저장소(GitHub)로 코드를 올리기 위해서는 다음 세 가지가 필수적입니다.

**1. 로컬 저장소 클론 (Clone)**
먼저, 해당 원격 저장소를 로컬 컴퓨터로 복사(클론)해야 합니다. 아직 로컬에 해당 저장소가 없다면 이 단계를 먼저 수행해야 합니다.

```bash
# 1단계: 원격 저장소를 로컬 디렉토리로 복사합니다. (아직 클론하지 않았다면 실행)
git clone https://github.com/songssam1004-cpu/260425_memory.git
cd 260425_memory
```

**2. 변경 사항 커밋 (Commit)**
로컬에서 파일을 수정한 후, 해당 변경 사항을 Git의 기록으로 저장소에 반영(커밋)해야 합니다. `git push`는 '커밋된 내용'을 원격으로 보낸다는 의미이므로, 반드시 커밋이 선행되어야 합니다.

```bash
# 2단계: 변경된 파일을 스테이징 영역에 추가합니다. (모든 변경 파일 추가 시)
git add .

# 3단계: 변경 사항을 로컬 저장소에 기록합니다. (커밋 메시지는 명확하게 작성해야 합니다.)
git commit -m "feat: [변경 내용에 대한 구체적인 설명]"
```

**3. 원격 저장소 연결 확인 (Remote Check)**
로컬 저장소가 원격 저장소(`origin`)를 올바르게 바라보고 있는지 확인합니다. 일반적으로 `git clone` 명령을 사용했다면 자동으로 설정됩니다.

```bash
# 원격 저장소 목록 확인 (origin이 있어야 합니다.)
git remote -v
```

### 📤 구체적인 Git Push 명령어

로컬에서 커밋을 완료했다는 가정 하에, 원격 저장소로 변경 사항을 업로드하는 명령어는 다음과 같습니다.

#### A. 최초 푸시 (새로운 브랜치 생성 시)

만약 로컬에서 새로 만든 브랜치를 원격으로 처음 올리는 경우 사용합니다.

```bash
# 현재 로컬 브랜치(예: main 또는 master)를 원격 저장소(origin)의 해당 브랜치로 푸시합니다.
git push -u origin <브랜치_이름>
```
*   **`<브랜치_이름>`**: 실제로 코드를 올리고 싶은 브랜치 이름 (예: `main` 또는 `master`)을 입력해야 합니다. `-u` 옵션은 로컬 브랜치가 원격 저장소의 해당 브랜치를 추적(tracking)하도록 설정합니다.

#### B. 이후 푸시 (기존 브랜치 업데이트 시)

이미 한 번 푸시한 적이 있고, 현재 작업 중인 브랜치에 추가 커밋이 있는 경우 사용합니다.

```bash
# 이미 설정된 원격 저장소로 로컬의 변경 사항을 푸시합니다.
git push
```
*   **참고**: 만약 `git push`만 입력했을 때 권한 오류가 발생한다면, GitHub 계정 인증(SSH 키 또는 Personal Access Token)이 올바르게 설정되어 있는지 확인해야 합니다.

### 💡 요약 실행 흐름 (Workflow Summary)

1.  **복제:** `git clone https://github.com/songssam1004-cpu/260425_memory.git`
2.  **작업 및 커밋:** 변경 사항 작업 후, `git add .` $\rightarrow$ `git commit -m "..."`
